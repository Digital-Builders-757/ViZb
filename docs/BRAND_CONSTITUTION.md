# Brand constitution — ViBE (Layer 1)

**Last updated:** March 23, 2026

**Non-negotiable brand laws.** For tokens, type scales, components, and examples, use **`docs/BRAND_SYSTEM.md`**. For channel-specific patterns, use **`docs/brand/`**.

---

## 1. Identity

ViBE is **young, cultural, energetic, inclusive, authentic, bold, editorial, movement-driven** — streetwear editorial meets community-first events.

**Failure test:** If a screen could be mistaken for generic SaaS, it fails.

---

## 2. Digital expression (summary)

| Law | Detail |
|-----|--------|
| **Dark-first** | Default experience is dark; light mode is out of scope until spec says otherwise. |
| **Sharp UI** | Zero-radius (or brand-defined radius) discipline per `BRAND_SYSTEM` — no soft generic cards-by-default. |
| **High contrast** | Readable, accessible contrast; neon gradient is for **brand moments** only (hero, key titles, primary CTAs). |
| **Typography** | **Space Grotesk** (`font-sans`) for UI chrome and body; editorial serif for headlines per `BRAND_SYSTEM`; mono for labels/metadata. |
| **Palette** | **ViBE Blue** (`--primary`), **cyan accent** family — canonical hex/tokens only from `BRAND_SYSTEM` §2. |

---

## 3. Content and copy

- Voice: direct, inclusive, credible — see **`docs/brand/VOICE_AND_MESSAGING.md`** and `BRAND_SYSTEM` §11.
- Event promos, landing copy, and social snippets should follow **`docs/brand/CONTENT_PATTERNS.md`** and **`docs/brand/EVENT_CREATIVE_RULES.md`** so the **content ↔ event flywheel** stays coherent.

---

## 4. What agents must do

Before shipping UI or user-facing copy:

1. Read **`docs/BRAND_CONSTITUTION.md`** (this file) + relevant **`docs/brand/*`**.
2. Run or self-apply **`/brand-check`** for visible changes.

---

## 5. Amendment process

Changing these laws requires updating **`docs/BRAND_SYSTEM.md`** in the same change set (or explicitly deferring detail there) and notifying in **`BRAND_STATUS.md`**.
