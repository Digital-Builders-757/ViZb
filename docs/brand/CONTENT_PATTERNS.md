# Content patterns — ViBE

**Last updated:** March 23, 2026

Structural patterns so **landing, product UI, email, and notifications** stay aligned. Voice detail: **`VOICE_AND_MESSAGING.md`**.

## Landing / marketing blocks

- **Hero:** value prop + single primary CTA (discover events, join waitlist, sign up).
- **Proof:** real communities, real events — numbers only if truthful.
- **Editorial grid:** event culture, not generic feature bullets.

## In-app (events)

- **Event card:** title, date/time, city, category badge, flyer thumb, clear state (RSVP / tickets / sold out).
- **Event detail:** who, when, where, flyer, ticket/RSVP CTA, organizer attribution.

## Email / notifications (roadmap)

- Subject lines: short, human, no spam patterns.
- Body: mirror in-app terminology (same event title, same date formatting).

## Sync rule

When copy changes, use **`/content-sync`** and update **`PRODUCT_STATUS_REPORT.md`** if the change affects roadmap messaging.
